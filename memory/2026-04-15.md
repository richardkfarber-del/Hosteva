
## 2026-04-15 17:54 EDT - 48-Hour Architecture Autopsy & Swarm Reset
- **Root Cause of Code Execution Failure (TTY/PTY Bug):** Background Python daemons (e.g., `auto_pipeline.py` or Temporal workers) executing OpenClaw via headless `subprocess.run` strip the TTY/PTY environment. This causes the `exec`, `edit`, and `browser` tools to hang or fail silently, breaking the Swarm's ability to edit code or test UIs.
- **Agent Hallucinations (Sycophancy):** Local quantized models degrade under complex JSON tool-call schemas and bloated contexts. Instead of throwing errors for missing paths or broken environment variables, they output plaintext narratives hallucinating successful file edits (e.g., Wasp claiming to fix logos without ever using the `edit` tool).
- **State Management & Orchestration Pivot:** `Temporal.io` is officially classified as overkill (heavy gRPC/boilerplate for local LLMs), and `LangGraph` scripts are too rigid. The new roadmap dictates a lightweight event-driven model: Redis (ephemeral state/pub-sub) + PostgreSQL (persistence) wrapped in a simple FastAPI REST interface. Agents will manage state via standard REST endpoints (`/state/update`).
- **Memory Unification (`pgvector`):** Dual-database stacks (Postgres + standalone VectorDBs like Chroma) create sync overhead. Wanda's memory will be unified into PostgreSQL using the `pgvector` extension to allow hybrid relational and semantic searches in a single SQL query. Flat-file parsing (`MEMORY.md`) is deprecated for long-term state persistence.
