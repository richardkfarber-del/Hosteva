# BUG-006: Gemini RAG Missing PostgreSQL Role Permissions

## Context
The LangGraph autopilot attempted to loop Iron Man and Vision directly back into Execution (Strike 1) without enforcing proper bug tracking. This ticket enforces the New Bug Process mandated in Sprint 5. The Gemini RAG endpoint currently lacks explicit PostgreSQL role restrictions.

## Requirements
Iron Man is formally required to write restrictive `GRANT SELECT` statements and Row-Level Security (RLS) policies into `database/schema.sql` to restrict the Gemini RAG endpoint strictly to a read-only role.

## Acceptance Criteria
- **Given** the Gemini RAG service queries the database,
- **When** the read-only role accesses the tables,
- **Then** `GRANT SELECT` statements enforce read-only access.
- **And** Row-Level Security (RLS) policies explicitly prevent unauthorized data mutation.
