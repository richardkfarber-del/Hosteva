# BUG-009: Dashboard BOLA & DOM XSS Vulnerabilities

## Context
Two critical security vulnerabilities have been identified during quality assurance:
1. **BOLA/IDOR Vulnerability (Backend):** The endpoint `GET /api/user/analytics` currently fetches data using `db.query(Host).first()` rather than scoping to the authenticated user's session.
2. **DOM-based XSS (Frontend):** The `templates/dashboard.html` file renders user input directly into the DOM using raw template string interpolation into `innerHTML`.

## Acceptance Criteria

### Iron Man (Backend)
- Given an authenticated user requests `GET /api/user/analytics`
- When the database query is executed
- Then the database query MUST filter by `current_user.id` (e.g., using `current_user.id` instead of `db.query(Host).first()`)
- And the endpoint MUST NOT return data belonging to any other user.

### Wasp (Frontend)
- Given a user viewing the dashboard
- When dynamic content is rendered into the DOM in `templates/dashboard.html`
- Then the frontend MUST use `textContent` OR sanitize the input using `DOMPurify` instead of directly setting `innerHTML` with raw strings
- And malicious scripts MUST NOT execute.
