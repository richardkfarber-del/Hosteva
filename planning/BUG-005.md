# BUG-005: UI Regression (Logo Rendering & Orphaned Integrations Nav)

## Domain
Design / Frontend Integration

## Status
**READY FOR EXECUTION**

## Context
A UI regression was introduced affecting the application logo rendering and leaving an orphaned navigation link for integrations. This impacts the user experience and visual brand consistency.

## Acceptance Criteria (Gherkin)

**Scenario:** Logo Rendering Fix
**Given** a user navigates to any page of the Hosteva application
**When** the frontend application renders the global header or navigation bar
**Then** the application must load and display the Hosteva logo at the correct dimensions without distortion
**And** the logo file path must resolve correctly without 404 errors.

**Scenario:** Orphaned Integrations Nav Cleanup
**Given** the user is viewing the main navigation menu
**When** the menu items are generated
**Then** the UI must not display any orphaned or broken "Integrations" links
**And** the integrations options must only appear in their correct, designated settings area.

## Definition of Done (DoD)
- [ ] Visual regression tests pass for the logo rendering in the header.
- [ ] E2E/Integration tests confirm the absence of orphaned links in the navigation tree.
- [ ] Code is formatted and linted properly.
- [ ] Mandatory automated test requirement: Playwright or headless UI automated tests must verify logo visibility and navigation structure.
